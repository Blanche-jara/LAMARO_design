# Espresso Machine Recipe Flow - 설계 문서

## 개요
에스프레소 머신 레시피 편집 플로우. Meticulous Home Inc. 앱을 참고하되, 아래 확정된 사항을 기반으로 설계.

## Stage 기반 프로파일 구조

### 핵심 개념
프로파일은 **Stage** 단위로 구성된다. 각 Stage는 이전 값에서 목표값까지 설정된 시간 동안 램프한다.

| Stage | 역할 | 필수 | 삭제 가능 |
|-------|------|------|----------|
| Stage 1 | Pre-infusion (0에서 목표값으로 램프) | O | X (duration=0으로 비활성) |
| Stage 2 | Extraction (Stage 1 목표에서 추출 목표로 램프) | O | X |
| Stage 3~10 | 추가 프로파일 변화 (사용자 정의) | X | O |

### Stage 파라미터
| 파라미터 | 설명 | 단위 | 범위 |
|---------|------|------|------|
| duration | 램프 시간 (Stage 1은 0 가능 = 비활성) | s | 0~30s |
| target | 목표 압력 또는 유량 | bar / ml/s | 0~15 / 0~10 |
| rampType | 커브 형태 | Linear / Exponential | — |

### 프로파일 동작
```
Stage 1 (5s, 3bar, Linear):   0~5s   → 0에서 3bar로 램프
Stage 2 (2s, 9bar, Expo):     5~7s   → 3에서 9bar로 램프
Stage 3 (10s, 6bar, Linear):  7~17s  → 9에서 6bar로 램프
(이후):                        17~40s → 6bar 유지 → maxShotTime까지
```

## 확정된 결정사항

| 항목 | 결정 | 근거 |
|------|------|------|
| 프로파일 모드 | **통일 모드** — 전체 레시피가 압력(bar) 또는 유량(ml/s) 중 하나 | Y축 스케일 단순화, UI 복잡도 감소 |
| 전환 방식 | **Stage 램프** — Stage 2의 rampType이 전환 커브를 결정 | 별도 전환 시간 불필요, Stage 시스템에 통합 |
| 온도 | **단일 고정 온도** | 그래프 코너에 배지로 표시 |
| 구현 범위 | **레시피 편집 UI + 프로파일 프리뷰 그래프** | 실시간 추출 화면은 2차 |
| Pre-infusion 시간 vs 압력 | **시간 우선** | 시간 초과 시 압력 무관하게 다음 Stage 진행 |
| Pre-infusion 생략 | **허용** — Stage 1 duration=0이면 건너뛰기 | 레시피에 따라 pre-infusion 없는 경우 존재 |
| 종료 조건 | **먼저 도달하는 조건 우선** | 최대시간=안전장치, 무게=정상 종료 |
| 그래프 X축 | **시간(s)만 지원** | 추출량은 종료조건으로만 사용 |
| Stage 제한 | **최소 2, 최대 10** | Stage 1(PI) + Stage 2(Ext)는 항상 존재 |

## 논리적 문제점 및 해결 방안 (총 12개)

### Q1. 압력 모드 vs 유량 모드 혼합
- **결정**: ❌ 통일 모드만 허용. 전체 레시피가 압력 또는 유량 중 하나.

### Q2. Stage 간 전환 방식
- **결정**: ✅ 각 Stage의 rampType과 duration이 전환을 정의. 별도 전환 파라미터 불필요.

### Q3. 추출 종료 조건 우선순위
- **결정**: ✅ 먼저 도달하는 조건이 우선.

### Q4. Pre-infusion 생략 가능 여부
- **결정**: ✅ Stage 1 duration=0이면 건너뛰기.

### Q5. 추출 온도 변화 방식
- **결정**: ❌ 단일 고정 온도만 지원.

### Q6. X축 = 추출량일 때 Pre-infusion 구간 표현 문제
- **결정**: ✅ X축은 시간(s)만 지원.

### Q7. 유량 모드에서의 압력 안전 제한
- **결정**: 하드웨어 레벨에서 최대 압력 제한. 소프트웨어에서는 경고 표시.

### Q8. 블루투스 저울 연결 실패 시 동작
- **결정**: 이번 범위 밖. 2차 구현 시 사용자 선택 팝업.

### Q9. Pre-infusion에서 유량 모드 사용 시 불안정성
- **결정**: 허용하되 경고 표시.

### Q10. 그래프: 설정 프로파일 vs 실시간 데이터
- **결정**: ❌ 이번 범위는 레시피 편집 + 프리뷰 그래프만.

### Q11. 레시피 저장 및 관리
- **결정**: SharedPreferences에 JSON으로 로컬 저장.

### Q12. 그래프 Y축 스케일 차이
- **결정**: 통일 모드이므로 모드에 따라 Y축 라벨과 범위를 동적으로 변경.

## 그래프 사양

```
Y축 (bar 또는 ml/s)
│
│ 15 ┤
│    │
│  9 ┤    S2         S3 ●───────── 마지막 Stage target 유지
│    │   ╱  ╲       ╱
│  6 ┤  ╱    ╲     ╱
│    │ ╱      ╲   ╱
│  3 ┤╱ S1     ╲─╱  S4
│    │          S3
│  0 ┤──┬──┬──┬──┬────── X축 (시간, s)
     0  S1 S2 S3 S4    maxShotTime
                        [93℃]  ← 온도 배지
                        [36g]   ← 종료 무게
```

### 그래프 색상
- **Stage 1** (녹색): Pre-infusion 구간
- **Stage 2+** (노란색): 하나의 연속 라인으로 결합, Stage 경계에 점 마커
- **Stage 경계선**: 수직 점선 + "S1", "S2" 라벨

### 그래프 요소
- **온도 배지**: 그래프 우측 상단 코너
- **종료 무게**: 별도 라벨
- **Stage 경계**: 점선 수직선 + 라벨
- **Stage 경계 점**: 노란색 원형 마커 (Stage 3+ 경계에만 표시)
- **Stage 1 duration=0**: 녹색 구간 없이 전체 노란색
