# Espresso Machine Recipe Flow - 설계 문서

## 개요
에스프레소 머신 레시피 편집 플로우. Meticulous Home Inc. 앱을 참고하되, 아래 확정된 사항을 기반으로 설계.

---

## 제어 요소

| # | 요소 | 설명 | 단위 | 범위 |
|---|------|------|------|------|
| 1 | 추출 타깃 압력/유량 | Extraction 구간에서 유지할 목표값 | bar 또는 ml/s | 0~15bar / 0~10ml/s |

## 확정된 결정사항

| 항목 | 결정 | 근거 |
|------|------|------|
| 프로파일 모드 | **통일 모드** — 전체 레시피가 압력(bar) 또는 유량(ml/s) 중 하나 | Y축 스케일 단순화, UI 복잡도 감소 |
| 전환 방식 | **램프 전환** — PI→Extraction 시 설정된 전환 시간에 걸쳐 부드럽게 변화 | 물리적으로 즉시 전환 불가, transitionTime 파라미터로 제어 |
| 온도 | **단일 고정 온도** | 그래프 코너에 배지로 표시 |
| 구현 범위 | **레시피 편집 UI + 프로파일 프리뷰 그래프** | 실시간 추출 화면은 2차 |
| Pre-infusion 시간 vs 압력 | **시간 우선** | 시간 초과 시 압력 무관하게 Extraction 진행 |
| Pre-infusion 생략 | **허용** — 시간=0이면 건너뛰기 | 레시피에 따라 pre-infusion 없는 경우 존재 |
| 종료 조건 | **먼저 도달하는 조건 우선** | 최대시간=안전장치, 무게=정상 종료 |
| 그래프 X축 | **시간(s)만 지원** | 추출량은 종료조건으로만 사용 |

## 논리적 문제점 및 해결 방안 (총 12개)

### Q1. 압력 모드 vs 유량 모드 혼합
- **문제**: Pre-infusion과 Extraction이 각각 독립적인 모드(압력/유량)를 가질 수 있는가?
- **결정**: ❌ 통일 모드만 허용. 전체 레시피가 압력 또는 유량 중 하나.
- **이유**: Y축 스케일이 구간별로 달라지는 문제 회피. UI 단순화.

### Q2. Pre-infusion → Extraction 전환 방식
- **문제**: Pre-infusion 끝 → Extraction 시작 시 목표값이 어떻게 변하는가?
- **결정**: ✅ 즉시 변경 (스텝). 그래프에서 수직선으로 표현.
- **이유**: 추가 파라미터(전환 시간) 불필요. 하드웨어가 물리적 최대 속도로 따라감.

### Q3. 추출 종료 조건 우선순위
- **문제**: 종료 무게(g)와 최대 시간(s)이 둘 다 설정된 경우?
- **결정**: ✅ 먼저 도달하는 조건이 우선.
  - 무게 도달 → 정상 종료
  - 최대 시간 도달 → 강제 종료 + 오류/경고 알림 ("설정 확인 필요")
- **이유**: 최대 시간은 안전장치이므로, 도달 시 "설정 오류 가능성" 경고가 적절.

### Q4. Pre-infusion 생략 가능 여부
- **문제**: Pre-infusion 시간을 0으로 설정하여 건너뛰는 것이 허용되는가?
- **결정**: ✅ 허용. 시간=0이면 그래프에서 녹색 구간 없이 전체 노란색.
- **이유**: 에스프레소 레시피에 따라 pre-infusion 없이 바로 추출하는 경우 존재.

### Q5. 추출 온도 변화 방식
- **문제**: 온도를 시간에 따라 변화시킬 것인가?
- **결정**: ❌ 단일 고정 온도만 지원.
- **이유**: UI 단순화. 향후 선형/커스텀 커브 확장 가능하도록 모델은 유연하게.

### Q6. X축 = 추출량일 때 Pre-infusion 구간 표현 문제
- **문제**: Pre-infusion 중에는 커피가 거의 나오지 않아 X축=추출량이면 구간이 안 보임.
- **결정**: ✅ X축은 시간(s)만 지원.
- **이유**: 레시피 편집 그래프에서는 시간이 자연스러움. 추출량은 종료 조건으로만.

### Q7. 유량 모드에서의 압력 안전 제한
- **문제**: 유량 모드에서 원두가 너무 곱게 갈려 압력이 과도하게 상승하면?
- **결정**: 하드웨어 레벨에서 최대 압력 제한 (12~15bar). 소프트웨어에서는 경고 표시.
- **향후**: 유량 자동 감소 로직 추가 가능.

### Q8. 블루투스 저울 연결 실패 시 동작
- **문제**: 종료 조건=무게인데 저울 연결이 끊기면?
- **결정**: 이번 범위 밖 (실시간 추출 미포함). 2차 구현 시 사용자 선택 팝업으로.
- **향후**: 경고 팝업 → "최대 시간으로 진행" vs "취소" 선택.

### Q9. Pre-infusion에서 유량 모드 사용 시 불안정성
- **문제**: Pre-infusion에서 포르타필터에 물이 차면서 유량이 급격히 변동.
- **결정**: 허용하되 경고 표시. 통일 모드이므로 유량 선택 시 pre-infusion도 유량.
- **향후**: PID 안정화는 하드웨어/펌웨어 레벨에서 처리.

### Q10. 그래프: 설정 프로파일 vs 실시간 데이터
- **문제**: 실시간 추출 화면도 구현 범위에 포함되는가?
- **결정**: ❌ 이번 범위는 레시피 편집 + 프리뷰 그래프만.
- **향후**: 2차에서 목표값(점선) + 실시간 데이터(실선) 오버레이.

### Q11. 레시피 저장 및 관리
- **결정**: SharedPreferences에 JSON으로 로컬 저장.
- **향후**: 머신 BLE 동기화, 커뮤니티 공유는 별도 구현.

### Q12. 그래프 Y축 스케일 차이
- **문제**: 압력(0~15bar)과 유량(0~10ml/s)은 스케일이 다름.
- **결정**: 통일 모드이므로 모드에 따라 Y축 라벨과 범위를 동적으로 변경.
  - 압력 모드: Y축 0~15 bar
  - 유량 모드: Y축 0~10 ml/s

## Pre-infusion 시간 vs 압력 충돌 규칙 (상세)

> **규칙**: 시간이 절대 우선이다.
>
> 예시: 시간=3s, 압력=9bar 설정
> - 3초 안에 9bar 도달 불가능한 경우:
>   1. 최대한 압력을 올리는 데 집중
>   2. 3초가 지나면 현재 압력 무관하게 자동으로 Extraction 진행
>   3. Extraction 목표값으로 즉시 전환
>
> 예시: 시간=5s, 압력=3bar 설정
> - 2초 만에 3bar 도달한 경우:
>   1. 나머지 3초 동안 3bar 유지
>   2. 5초가 지나면 Extraction 진행
>
> **그래프 표현**: Pre-infusion 구간에서 0에서 목표값까지 Linear 또는 Exponential 커브로 상승

## 그래프 사양

```
Y축 (bar 또는 ml/s)
│
│ 15 ┤
│    │
│  9 ┤              ●─────────●──── waypoint 커브 (노란색, 점 표시)
│    │            ╱              ╲
│  7 ┤          ╱                  ●─── 마지막 waypoint까지 유지
│    │        ╱
│  3 ┤ ~~~~~~┤  PI (녹색)     Transition (회색 점선)
│    │       │
│  0 ┤───────┬─┬────────────────── X축 (시간, s)
     0    PI시간 +전환             최대시간
                                   [93℃]  ← 온도 배지
                                   [36g]   ← 종료 무게
```

### 4구간 구성
1. **Pre-infusion** (녹색 영역): 0 → PI target, Linear 또는 Exponential 커브
2. **Transition** (회색 점선): PI target → Extraction target, 설정된 전환 시간에 걸쳐 램프
3. **Extraction** (노란색 영역): Extraction target 유지 또는 변곡점에 따라 변화
4. **Waypoints** (노란색 점): Extraction 내 시간별 목표값 변경점, 각각 Lin/Exp 커브 선택 가능

### 그래프 요소
- **온도 배지**: 그래프 우측 상단 코너
- **종료 무게**: 별도 라벨
- **PI 종료선**: 점선 수직선 + "PI 5.0s" 라벨
- **Waypoint 점**: 노란색 원형 마커 (변곡점 위치에만 표시)
- **preInfusionTime=0**: 녹색/전환 구간 없이 전체 노란색

### Extraction 변곡점 (Waypoints)
- Extraction 시작 후 N초에 목표값을 변경하는 포인트
- 각 waypoint에 개별 커브 타입(Linear/Exponential) 설정 가능
- waypoint 간 보간: 선택된 커브 타입에 따라 직선 또는 지수 커브
- 마지막 waypoint 이후: 해당 목표값을 maxShotTime까지 유지
- waypoint 없으면: extractionTarget을 끝까지 수평 유지
